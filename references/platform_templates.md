# 四平台选题适配模板

本文件用于把同一个热点转成不同平台的内容表达：公众号、小红书、微信视频号、抖音。

## 平台定位

| 平台 | 内容形态 | 核心目标 | 主要指标 |
|---|---|---|---|
| 公众号 | 长文/深度分析/教程 | 建立专业信任、沉淀搜索长尾 | 打开率、读完率、转发、收藏 |
| 小红书 | 图文卡片/短笔记 | 场景种草、收藏、搜索发现 | 点击率、收藏率、评论、搜索曝光 |
| 微信视频号 | 30-90 秒口播/观点短视频 | 熟人社交传播、专业观点表达 | 完播、点赞、转发给朋友/群、评论 |
| 抖音 | 15-60 秒强钩子短视频 | 注意力抓取、完播、互动 | 前3秒留存、完播率、评论、转发 |

## 公众号模板：wechat_article

### 评分模型

| 维度 | 权重 | 说明 |
|---|---:|---|
| 人设匹配度 | 25% | 是否符合 AI/科技/效率/产品/创业读者 |
| 信息增量 | 20% | 是否有新事实、新判断、新方法 |
| 实操价值 | 20% | 是否能输出教程、清单、案例 |
| 观点深度 | 15% | 是否能形成独立判断 |
| 时效性 | 10% | 是否最近 30 天内仍有讨论价值 |
| 转发价值 | 10% | 是否适合转给同事/同行 |

### 输出字段

```markdown
## 公众号选题
- 选题主题：
- 推荐标题：
- 目标读者：
- 核心观点：
- 文章结构：
  1. 背景/热点
  2. 关键变化
  3. 影响分析
  4. 实操建议
  5. 风险/边界
- 开头钩子：
- 金句：
- 原文链接：
- 评分：
```

## 小红书模板：xhs

### 评分模型

| 维度 | 权重 | 说明 |
|---|---:|---|
| 封面点击潜力 | 20% | 首图标题是否短、明确、有结果感 |

| 评论触发 | 15% | 是否能引发“求工具/求教程/我也遇到” |

  1. 结果/痛点

  4. 步骤二
  5. 案例/对比
  6. 避坑
  7. 总结/收藏引导

## 标题改写规则

同一热点在四个平台的表达要明显不同。

示例热点：Claude Code Routines

| 平台 | 标题表达 |
|---|---|
| 公众号 | Claude Code Routines 来了：AI Agent 从“会写代码”走向“会定时干活” |
| 小红书 | 我想搭一个“每天自动找选题”的 AI 助手 |
| 微信视频号 | 以后你每天重复做的工作，可能真的可以交给 AI 了 |
| 抖音 | AI 已经不是聊天机器人了，它开始自己定时干活了 |

## 风险控制

- 不使用“必爆、稳赚、替代所有人、马上失业”等绝对化表达。
- 不编造数据；没有来源时写“据公开信息/据热榜显示/从讨论热度看”。
- 小红书和抖音避免过度焦虑营销。
- 视频号适合稳健表达，避免过度标题党。
- 公众号必须补充事实来源和边界条件。
