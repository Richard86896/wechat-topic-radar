# 过滤规则

本文件用于每日公众号选题的前三层过滤：关键词过滤 → AI 模糊过滤 → 实操价值过滤。最终评分仍以 SKILL.md 的三步法为准。

## 1. 关键词硬排除

命中以下关键词且无法连接到 AI 出海 / AI 编程 / Agent / 新工具 / AI 副业时，默认排除：

```text
明星
离婚
恋情
综艺
演唱会
体育赛果
天气
地震速报
交通事故
股价涨跌
汇率
节日祝福
抽奖
优惠券
带货
直播间
房价
彩票
八卦
```

## 2. AI 无关排除

以下情况直接排除：

- 没有 AI、模型、Agent、自动化、开发者工具、跨境增长、内容生产、开源项目等连接点；
- 只能写成社会新闻复述，无法提供判断或落地解释；
- 读者看完没有下一步动作；
- 需要强情绪、焦虑、站队才能传播。

## 3. 标题党和不可信线索排除

命中以下任一项，默认不进入主推荐，只能进观察池：

- 标题含“震惊、炸裂、全网疯传、彻底变天、吊打、碾压、封神、史上最强”等，但无权威来源；
- 只有单一截图、单一 X 爆料或二手转述；
- 涉及 benchmark、融资、法律、安全事故，但无官方/论文/法院/权威媒体来源；
- 标题明显隐藏关键信息，靠“没想到、竟然、背后真相”制造悬念。

## 4. 保留和优先信号

优先保留：

- AI 编程、Agent、workflow、IDE、CLI、开发者工具；
- OpenAI、Anthropic、Google、Meta、GitHub、Cursor、Claude Code、Codex 等大厂/头部工具；
- 国产模型和应用生态，但涉及性能/成本/击败谁时必须核验；
- 开源项目、GitHub Trending、真实 release note；
- AI 出海、跨境获客、AI SEO、内容增长、自动化流程；
- 能输出表格、清单、框架、实操步骤、避坑建议的题。

## 5. 实操价值过滤

通过以下问题判断是否保留：

1. 读者能不能据此少走弯路？
2. 能不能给出“值不值得关注”的判断？
3. 能不能落到工具选择、操作步骤、风险边界或机会判断？
4. 能不能沉淀为也船长AI长期内容资产？

至少满足 2 条才进入评分。

## 6. 观察池规则

以下内容不直接淘汰，但默认进入观察池：

- 新概念很有趋势感，但来源不足；
- 传播潜力高，但事实风险高；
- 与账号方向有关，但暂时没有足够素材；
- 适合小红书/短视频，但不适合公众号长文。
