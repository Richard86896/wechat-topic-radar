# 语言安全词规则

本文件定义文章中需要避免的不当表述及其推荐替换方案。

---

## 一、确定性过强用语

### 1.1 原因推断类

| 避免用语 | 问题 | 推荐替换 |
|----------|------|----------|
| "XX选择在AI爆发期卸任" | 过度确定原因 | "XX此时卸任，原因多方面" |
| "XX显然已经落后" | 主观判断 | "XX在XX浪潮中相对谨慎" |
| "这明显是XX" | 过度确定 | "这可能是XX" |
| "XX注定会XX" | 绝对化 | "XX可能会XX" |

### 1.2 未来预测类

| 避免用语 | 问题 | 推荐替换 |
|----------|------|----------|
| "苹果将推出XX" | 过度确定 | "苹果可能探索XX方向" |
| "XX将彻底改变XX" | 绝对化 | "XX可能深刻影响XX" |
| "这标志着XX时代的开始" | 过度宏大 | "这可能预示着XX趋势" |
| "必将XX" | 绝对化 | "有望XX" |

---

## 二、数据表述规范

### 2.1 精确数据

| 情形 | 避免 | 推荐 |
|------|------|------|
| 波动值 | "$4,000亿" | "接近$4,000亿" 或 "约$4,000亿" |
| 估算值 | "15万员工" | "约15万员工" |
| 比例 | "增长100%" | "增长约100%" 或 "翻倍" |

### 2.2 数据来源标注

- 财务数据：标注"来源于公开财报"或"公开报道"
- 市场数据：标注"来源：XX机构"
- 用户数据：标注"来源：XX报告"

---

## 三、高管与人事表述

### 3.1 职位变动

| 避免 | 推荐 |
|------|------|
| "被任命为首席XX官"（未经证实） | "担任XX职务" |
| "XX是XX之父/之母"（过度归因） | "XX是XX的核心贡献者" |
| "接替XX成为新CEO"（未生效） | "将于XX日期接任CEO" |

### 3.2 技术归属

| 避免 | 推荐 |
|------|------|
| "XX主导了XX芯片研发"（非其负责） | "XX参与XX项目" |
| "XX是AI战略负责人"（非其职） | "XX负责XX业务" |

---

## 四、时间事件表述

### 4.1 时间准确

| 避免 | 推荐 |
|------|------|
| "库克最近宣布退休" | "库克于2026年4月宣布将于9月退休" |
| "苹果3月刚宣布XX"（未经证实） | "苹果在XX时间点（根据公开报道）" |

### 4.2 事件状态

| 避免 | 推荐 |
|------|------|
| "库克已退休"（未发生） | "库克宣布将于9月退休" |
| "XX已完成交接"（未发生） | "XX将于XX日期完成交接" |

---

## 五、行业对比表述

### 5.1 竞争态势

| 避免 | 推荐 |
|------|------|
| "苹果AI明显落后" | "苹果在AI领域相对谨慎" |
| "XX已经超越XX" | "XX在某些方面表现突出" |
| "XX将彻底打败XX" | "XX可能对XX形成挑战" |

### 5.2 能力评估

| 避免 | 推荐 |
|------|------|
| "Apple Intelligence已成熟" | "Apple Intelligence正在推进中" |
| "XX拥有最领先的AI" | "XX在AI领域处于领先地位" |
| "XX已完全落后" | "XX面临挑战" |

---

## 六、禁用词汇清单

以下词汇在未经证实情况下严禁使用：

- "内部消息"、"据知情人士透露"
- "必然"、"注定"、"一定会"
- "彻底失败"、"全面落后"
- "官方宣布"（非官方来源）
- "权威人士"（匿名）

---

## 七、表述自检清单

生成文章后检查：

- [ ] 原因推断是否加了"可能"、"或许"等限定词
- [ ] 未来预测是否加了"可能"、"预期"等限定词
- [ ] 财务数据是否加了"约"、"接近"等限定词
- [ ] 高管归因是否准确（通过 WebSearch 实时验证）
- [ ] 时间事件是否准确（通过 WebSearch 实时验证）
- [ ] 行业对比是否保持中性（避免过度贬低或吹捧）
- [ ] 是否使用了禁用词汇
